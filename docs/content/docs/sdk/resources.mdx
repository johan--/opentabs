Resources expose read-only data from a web application to AI agents. When an agent calls `resources/read`, your resource's `read()` function runs in the browser page context with full access to the page's cookies, local storage, and JavaScript globals.

## Import

```typescript
import { defineResource } from '@opentabs-dev/plugin-sdk';
import type { ResourceDefinition, ResourceContent } from '@opentabs-dev/plugin-sdk';
```

## `defineResource(config)`

Type-safe factory function that returns a `ResourceDefinition`. Identity function — provides type checking with no runtime behavior.

```typescript
function defineResource(config: ResourceDefinition): ResourceDefinition;
```

## `ResourceDefinition`

```typescript
interface ResourceDefinition {
  uri: string;
  name: string;
  description?: string;
  mimeType?: string;
  read(uri: string): Promise<ResourceContent>;
}
```

## `ResourceContent`

## URI Prefixing

Resource URIs are automatically prefixed with `opentabs+<plugin>://` for MCP registration. A resource with URI `slack://channels` in a plugin named `slack` becomes `opentabs+slack://slack://channels` at the MCP level. You define just the local URI.

## Example

```typescript
import { defineResource, fetchJSON } from '@opentabs-dev/plugin-sdk';
import type { ResourceContent } from '@opentabs-dev/plugin-sdk';

export const channelList = defineResource({
  uri: 'slack://channels',
  name: 'Channel List',
  description: 'All Slack channels the user has access to.',
  mimeType: 'application/json',

  async read(uri: string): Promise<ResourceContent> {
    const data = await fetchJSON<{ channels: Array<{ id: string; name: string }> }>(
      '/api/conversations.list?types=public_channel,private_channel',
    );

    return {
      uri,
      text: JSON.stringify(data.channels, null, 2),
      mimeType: 'application/json',
    };
  },
});
```

### Registering on the Plugin Class

Assign your resources to the `resources` property on your plugin class:

```typescript
import { OpenTabsPlugin } from '@opentabs-dev/plugin-sdk';
import type { ToolDefinition, ResourceDefinition } from '@opentabs-dev/plugin-sdk';
import { channelList } from './resources/channel-list.js';
import { sendMessage } from './tools/send-message.js';

class SlackPlugin extends OpenTabsPlugin {
  readonly name = 'slack';
  // ... other properties
  readonly tools: ToolDefinition[] = [sendMessage];
  readonly resources: ResourceDefinition[] = [channelList];

  async isReady(): Promise<boolean> {
    // ...
  }
}

export default new SlackPlugin();
```

## Dispatch Flow

Resource reads follow the same dispatch pipeline as tool calls:

1. AI agent calls `resources/read` with the prefixed URI
2. MCP server strips the prefix and forwards to the Chrome extension via WebSocket
3. Extension routes to the correct tab's adapter
4. Adapter calls `read(uri)` in the page context
5. Result flows back through the chain

Unlike tool dispatch, resource reads do not support progress reporting or lifecycle hooks.
