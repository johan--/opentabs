The OpenTabs MCP server runs on `localhost:9515` by default and exposes HTTP endpoints for MCP clients (AI agents), a WebSocket endpoint for the Chrome extension, and admin endpoints for server management.

## HTTP endpoints

| Endpoint | Method(s) | Auth | Description |
|---|---|---|---|
| `/mcp` | `POST`, `GET`, `DELETE` | Bearer token | MCP Streamable HTTP transport — AI clients connect here. |
| `/ws` | WebSocket upgrade | `Sec-WebSocket-Protocol` | Chrome extension connection (full-duplex JSON-RPC). |
| `/ws-info` | `GET` | Bearer token or rate-limited | Returns WebSocket URL and auth secret for extension bootstrap. |
| `/health` | `GET` | None | Server health, plugin status, and metrics. |
| `/reload` | `POST` | Bearer token, rate-limited | Trigger config reload and plugin re-discovery. |
| `/extension/reload` | `POST` | Bearer token, rate-limited | Signal the Chrome extension to reload. |

## /mcp — MCP Streamable HTTP

The primary endpoint for MCP clients. Implements the [MCP Streamable HTTP transport](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http).

- **POST**: Send an MCP request (initialize, tools/list, tools/call, etc.)
- **GET**: Retrieve from an existing session (server-sent events)
- **DELETE**: Close a session

### Headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | Yes (if secret configured) | `Bearer <secret>` where `<secret>` is the 64-char hex token from `~/.opentabs/config.json`. |
| `mcp-session-id` | After initialization | Session UUID returned by the server on the initial `initialize` request. |

### Session lifecycle

1. Client sends a `POST` with an `initialize` request body (no session ID)
2. Server creates a new session and returns an `mcp-session-id` header
3. Subsequent requests include the `mcp-session-id` header
4. Each session has its own tool list and state
5. `DELETE` closes the session

Multiple concurrent MCP clients are supported — each gets an independent session.

## /ws — WebSocket

The Chrome extension connects here for full-duplex JSON-RPC 2.0 communication. Only one active extension connection is allowed — a new connection replaces the previous one.

### Authentication

The extension authenticates via the `Sec-WebSocket-Protocol` header (not URL parameters, to keep secrets out of logs):

```
Sec-WebSocket-Protocol: opentabs, <secret>
```

The server echoes `opentabs` as the accepted subprotocol.

### Limits

| Setting | Value |
|---|---|
| Max payload | 10 MB |
| Concurrent connections | 1 (new replaces old) |

## /ws-info — WebSocket bootstrap

Returns the WebSocket URL and auth secret so the extension can connect. Used by the extension's side panel during initial setup.

```json
{
  "wsUrl": "ws://localhost:9515/ws",
  "wsSecret": "a1b2c3d4..."
}
```

The `wsSecret` field is only included if a secret is configured. Rate-limited to 10 requests/minute as a fallback when Bearer auth fails.

## /reload — Config reload

Triggers a full config reload and plugin re-discovery. Requires Bearer token authentication. Rate-limited to 10 requests/minute.

```bash
curl -X POST http://localhost:9515/reload \
  -H "Authorization: Bearer <secret>"
```

Returns:

```json
{
  "ok": true,
  "plugins": 1,
  "durationMs": 45
}
```

## /extension/reload — Extension reload signal

Sends a reload signal to the connected Chrome extension. Requires Bearer token authentication. Rate-limited to 10 requests/minute. Returns 503 if the extension is not connected.

```bash
curl -X POST http://localhost:9515/extension/reload \
  -H "Authorization: Bearer <secret>"
```

## Authentication

All authenticated endpoints use the same shared secret from `~/.opentabs/config.json` (the `secret` field — a 64-char hex string auto-generated on first server run).

| Transport | Method |
|---|---|
| HTTP endpoints (`/mcp`, `/reload`, `/extension/reload`) | `Authorization: Bearer <secret>` header |
| WebSocket (`/ws`) | `Sec-WebSocket-Protocol: opentabs, <secret>` header |
| `/ws-info` | Bearer token, or rate-limited access (10 req/min) |
| `/health` | No authentication required |

When no secret is configured (field missing or null), authentication checks are skipped and all endpoints are open.

## CORS

The server rejects all HTTP requests that include a browser `Origin` header, except those from `chrome-extension://` origins. MCP clients (Claude Code, Cursor) run as native processes and don't send Origin headers, so this blocks browser-based requests without affecting legitimate clients. This prevents DNS rebinding attacks on the localhost server.

## Rate limiting

Admin endpoints (`/ws-info`, `/reload`, `/extension/reload`) are rate-limited to **10 requests per minute** using a sliding window. When the limit is exceeded, the server returns HTTP 429 with a `Retry-After: 60` header.

## WebSocket protocol

The extension and server communicate over JSON-RPC 2.0. Key message types:

### Server → Extension

| Method | Type | Description |
|---|---|---|
| `sync.full` | Notification | Full plugin list — sent on connection and after reload. |
| `plugin.update` | Notification | Single plugin updated (file watcher detected changes). |
| `tool.dispatch` | Request | Dispatch a tool call to the correct tab's adapter. |
| `tool.invocationStart` | Notification | Tool execution started (for side panel UI animation). |
| `tool.invocationEnd` | Notification | Tool execution completed (for side panel UI animation). |
| `extension.reload` | Request | Signal the extension to reload. |
| `pong` | Notification | Reply to keepalive ping. |

### Extension → Server

| Method | Type | Description |
|---|---|---|
| `tab.syncAll` | Notification | Full tab-to-plugin state mapping — sent on connection. |
| `tab.stateChanged` | Request | Plugin tab state changed (closed/unavailable/ready). |
| `config.getState` | Request | Fetch current config state (used by side panel). |
| `config.setToolEnabled` | Request | Enable or disable a single tool. |
| `config.setAllToolsEnabled` | Request | Enable or disable all tools for a plugin. |
| `ping` | Notification | Keepalive ping. |

## Tool dispatch

When an MCP client calls a tool:

1. Server validates the input against the tool's JSON Schema (pre-compiled with Ajv)
2. Server sends a `tool.dispatch` JSON-RPC request over WebSocket
3. Extension finds the best matching tab by URL pattern and ranking
4. Extension executes the adapter's `handle()` function via `chrome.scripting.executeScript` in MAIN world
5. Result flows back: Extension → WebSocket → MCP Server → MCP Client

### Timeouts

| Stage | Timeout | Description |
|---|---|---|
| Server dispatch | 30 seconds | Server rejects the tool call if no response from extension. |
| Extension script execution | 25 seconds | Extension kills the script — deliberately shorter than server timeout to guarantee a response. |

### Concurrency

Up to 5 tool calls can run concurrently per plugin. Additional calls are queued.

## MCP client configuration

### Claude Code

Add to your project's `.mcp.json` or `~/.claude.json`:

```json
{
  "mcpServers": {
    "opentabs": {
      "type": "streamable-http",
      "url": "http://localhost:9515/mcp",
      "headers": {
        "Authorization": "Bearer <secret>"
      }
    }
  }
}
```

Replace `<secret>` with the value from `~/.opentabs/config.json`.

### Cursor

Add to your MCP settings:

```json
{
  "mcpServers": {
    "opentabs": {
      "url": "http://localhost:9515/mcp",
      "headers": {
        "Authorization": "Bearer <secret>"
      }
    }
  }
}
```

### Generic MCP client

Any MCP client that supports Streamable HTTP can connect:

- **URL**: `http://localhost:PORT/mcp` (default port 9515)
- **Auth**: `Authorization: Bearer <secret>` header
- **Transport**: Streamable HTTP (POST/GET/DELETE)

## Default port

The server listens on port **9515** by default. Override with the `PORT` environment variable:

```bash
PORT=3000 bun --hot platform/mcp-server/dist/index.js
```

Or use the CLI:

```bash
opentabs dev --port 3000
```
